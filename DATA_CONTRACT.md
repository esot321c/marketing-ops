# Data Contract

Marketing-Ops separates reusable system files from user-owned marketing context.

## System Layer

These files are reusable project logic and may be updated by upstream changes:

- `apps/dashboard/`
- `skills/`
- `templates/`
- `.codex-plugin/plugin.json`
- `README.md`
- `DATA_CONTRACT.md`
- `package.json`

## Public Example Layer

These files are safe to commit and share:

- `data.example/tenants/*.json`
- `data.example/setup/*.json`
- anonymous campaigns
- anonymous content examples
- template account inventories

## User Configuration Layer

These files personalize the system for a specific company, founder, or marketing motion:

- `data/brands/*.md`
- `data/tenants/*.json`
- `data/accounts/*.json`
- `data/campaigns/*.md`
- `data/competitors/*.md`
- `data/keywords/*.md`
- `data/voice/*.md`
- `data/setup/*/` (per-tenant Init state, intake, voice, and the tenant's **design system**: `design-system/tokens.json` + `design-system/previews/*.html`)
- `strategy/*.md` (optional, user-created local folder)
- `docs/*` (optional, user-created local folder for your own architecture notes and plans)

These private folders do not ship with the repo. You create them locally during init; they are gitignored and never committed. See the `marketing-setup` skill for how to create them.

Per-tenant design systems are user-owned data, not system code. The app ships **generic, reusable** components (`apps/dashboard/src/design-system/`); each tenant's brand tokens and `@dsCard` previews live under `data/setup/<tenant>/design-system/` and are served to the dashboard at runtime.

Agents may edit these files when the user asks them to update strategy, content, voice, or planning context.

## Generated Marketing Work

These files are generated by workflows and should be treated as user-owned output:

- `data/generated/`
- `output/`
- exported carousels
- exported scripts
- publishing queues
- analytics reports

## Local Tooling

These files should not be committed:

- `node_modules/`
- `.next/`
- `.env`
- `.env.local`
- local connector credentials

## Safety Rules

1. Do not publish externally without explicit user approval.
2. Do not send messages, outreach, DMs, or emails from Marketing-Ops.
3. Do not manage contacts, accounts, deal stages, or follow-up cadences here.
4. Hand off sales-ready context to your CRM instead of tracking it in Marketing-Ops.
5. Keep marketing source material and generated drafts reviewable as local files.
